# Changelog: AI Spend CFO

All notable changes to this product are recorded here. Format follows Keep a Changelog; entries are grouped by batch. House rule observed: no em dashes.

## [Unreleased]

### Added
- Building pack generated from DISCOVERY.md (batch B-pack, 2026-06-09): `CLAUDE.md` (project operating rules), `DECISIONS.md` (D1 to D18 plus controls and deferred non-goals), `BATCH_PLAN.md` (batches B0 to B5 with bounded file lists and the session prompt template), `SESSION_LOG.md`, and this `CHANGELOG.md`.
- Project scaffold and toolchain (batch B0, 2026-06-09): Next.js 16 (App Router) on React 19 with the approved dependency set pinned and lockfile committed; TypeScript (strict), ESLint flat config (`eslint-config-next` 16 native), and Vitest test runner all proven green on the untouched tree. Added the canonical usage and pricing schema (`lib/types.ts`), the deterministic compute barrel (`lib/metrics/index.ts`), a seed test, a placeholder landing page, and project config (`.npmrc`, `.gitignore`, `.env.example`, `README.md`). Git initialized with a genesis commit (not yet pushed). Toolchain decisions recorded as D22.
- Static credible demo (batch B1, 2026-06-09, D26): the first product surfaces, built on Tailwind v4 to the audit-grade-ledger design language (D20). A reproducible seeded generator (`data/generate-northstar.mjs`) produces the labeled-synthetic Northstar dataset (`data/northstar.json`, 558 events, closed month May 2026, costs derived from tokens). A pure read-only aggregation layer (`lib/metrics/aggregate.ts`, with `aggregate.test.ts`) computes spend composition and waste indicators. The cached hero CFO memo (`data/precomputed-memo.json`, D3, no live call) renders as a board-grade typeset document with dollar-backed flags and a visible in-code needs-review exclusion (C1 honesty stance), and a test reconciles every memo figure against the computed aggregates (C2 precursor). New surfaces: landing (`app/page.tsx`, leads with the memo), dashboard (`app/dashboard/page.tsx`, KPIs + Recharts-direct charts), memo (`app/memo/page.tsx`), and components (`site-nav`, `sample-banner`, `kpi-card`, `charts`, `cost-drivers-table`, `memo-view`). Synthetic data is labeled sample everywhere. All gates green: build, lint, typecheck, Vitest (17), Playwright (8).

### Changed
- Plan amended (2026-06-09) for a skill-driven build: added D19 (skills mapped to batches), D20 (institutional-finance-with-modern-execution design language), and D21 (memo PDF produced by a bespoke authored `cfo-memo-pdf` skill). BATCH_PLAN.md gained a skills mapping, per-batch skill annotations, and a new batch B5 (author the PDF skill); the old export batch became B6.
- Plan amended again (2026-06-09, post-B0): dropped `@tremor/react` in favor of Tailwind CSS plus Recharts-direct (D23); made "commit and push after every batch" and "Playwright as the standing UI harness" explicit project rules (D24). Inserted batch B0.5 (tooling switch plus git remote) before B1, updated CLAUDE.md's tech stack and per-batch discipline, and reconciled stale stack references. No product code changed.
- Playwright harness hardened (batch B1, 2026-06-09, D26): e2e now runs against a production server (`npm run build && npm run start`; `PW_DEV=1` opts back to dev) with `workers: 1`. The dev server's per-route Turbopack compile under parallel workers exceeded the test budget; the production, serial run is fast (~17s) and reliable and mirrors Vercel. Verification tooling only, no product change.
- Tooling switch executed (batch B0.5, 2026-06-09, D25): removed `@tremor/react` and deleted `.npmrc` (clean reinstall, no peer override; `npm ci` 440 packages, zero ERESOLVE warnings). Set up Tailwind CSS v4 (`tailwindcss` + `@tailwindcss/postcss`, CSS-first: `postcss.config.mjs`, `app/globals.css` with `@theme` tokens imported in the layout, no `tailwind.config.ts`) and converted the placeholder landing page to Tailwind. Moved Recharts to its v3 line (`3.8.1`). Installed Playwright (`@playwright/test` 1.60.0 + Chromium) as the standing UI harness with `playwright.config.ts` and an `e2e/smoke.spec.ts` that proves Tailwind actually compiled. All gates green: build, lint, typecheck, Vitest, and the Playwright smoke. Pushed to `origin/main`.
